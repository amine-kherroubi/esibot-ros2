## 📂 Repository Structure

This repository follows the standard ROS 2 workspace structure. Understanding this layout is crucial for knowing where to put your code.

```text
robot_ws/                     # The main workspace folder
├── src/                      # 📦 SOURCE CODE (This is the Git Repository)
│   ├── esibot_description/   # Robot Blueprints (URDF, Meshes)
│   ├── esibot_gazebo/        # Simulation (Gazebo World, Plugins)
│   ├── esibot_bringup/       # (To be created) when we go to hardware
│   ├── esibot_driver/        # (To be created)  Drivers 
│   ├── esibot_sensors/      # (To be created) Ultrasonic/Lidar logic
│   ├── esibot_navigation/   # (To be created) Nav2 & SLAM configs
│   └── esibot_camera/       # (To be created) Vision nodes
│
├── build/               # ⚙️ COMPILED FILES (Generated by colcon) not commited
├── install/             # 📍 INSTALLATION OVERLAY (Generated by colcon) not commited 
└── log/                 # 📝 LOGS (Generated by colcon) not commited
```

### 📁 Package Descriptions

#### 1. `esibot_description`
**Purpose:** Defines what the robot *is*.
*   **URDF/Xacro Files:** The mathematical description of the robot's shape (chassis, wheels, sensors).
*   **Meshes:** to do : 3D models (STL/DAE) for visualisation.
*   **RViz Configs:** to do : Saved settings for how we view the robot.


#### 2. `esibot_gazebo`
**Purpose:** Simulation 
*   **World Files:** The virtual environment (walls, floor, obstacles).
*   **Launch Files:** Scripts to start the simulation with the robot model loaded.
*   **Plugins:** Configurations that give the simulated robot physics and sensors (e.g., differential drive plugin).

## 🚀 Getting Started

### Prerequisites
Ensure you have followed the initial setup guide:
1.  Ubuntu 24.04
2.  ROS 2 Jazzy 
3.  Colcon & Rosdep

### Installation & Build

1.  **Clone the repository** (if starting fresh):
    ```bash
    mkdir -p ~/robot_ws/src
    cd ~/robot_ws/src
    git clone https://github.com/amine-kherroubi/robot.git .
    ```

2.  **pull the repository** (if already cloned):
    ```bash
    cd ~/robot_ws/src
    git pull origin main
    ```

3.  **Install Dependencies** (Safety check):
    ```bash
    cd ~/robot_ws
    rosdep install --from-paths src --ignore-src -r -y
    ```

4.  **Build the Workspace**:
    ```bash
    cd ~/robot_ws
    colcon build --symlink-install
    ```

5.  **Source the Workspace**:
    ```bash
    source ~/.bashrc
    ```

## 🎮 Usage

### Launching the Simulation
This starts the virtual robot in a Gazebo world.
```bash
ros2 launch esibot_gazebo sim.launch.py
```
