# EsiBot Project

**Autonomous Differential Drive Robot**
*Inspired by TurtleBot3 Burger | ROS 2 Jazzy | Gazebo Simulation*

---

## Abstract
EsiBot is an autonomous mobile robotics project based on a differential drive architecture, inspired by the TurtleBot3 Burger platform. Designed for educational and research purposes at the École nationale Supérieure d'Informatique (ESI), the system integrates a Raspberry Pi 4 running ROS 2 and an ESP32-CAM microcontroller. The platform features a custom 3D-printed chassis, a rotating ultrasonic radar system (an HC-SR04 sensor mounted on a SG90 servo) for obstacle detection, and a differential drive mechanism. The software stack leverages `slam_toolbox` for Simultaneous Localization and Mapping (SLAM) and the `Nav2` framework for autonomous path planning and navigation.

## Repository Structure

This repository follows the standard ROS 2 workspace architecture to modularize the system's functional components.

```text
robot_ws/
├── src/
│   ├── esibot_description/   # Robot kinematics, URDF, and visualization meshes
│   ├── esibot_gazebo/        # Simulation environments, Gazebo worlds, and plugins
│   ├── esibot_bringup/       # Hardware drivers and core initialization launch files
│   ├── esibot_sensors/       # Ultrasonic radar logic and sensor data publishing
│   ├── esibot_navigation/    # Nav2 and slam_toolbox configurations
│   └── esibot_camera/        # Computer vision nodes and ESP32-CAM stream processing
│
├── build/                    # Compiled binaries (Generated by colcon, untracked)
├── install/                  # Installation overlay (Generated by colcon, untracked)
└── log/                      # Build and execution logs (Generated by colcon, untracked)
```

## Getting Started

### System Requirements
1.  Ubuntu 24.04 LTS
2.  ROS 2 Jazzy
3.  Colcon build system and Rosdep

### Installation and Build Instructions

1.  **Clone the Repository:**
    ```bash
    mkdir -p ~/robot_ws/src
    cd ~/robot_ws/src
    git clone [https://github.com/amine-kherroubi/robot.git](https://github.com/amine-kherroubi/robot.git) .
    ```

2.  **Update an Existing Repository:**
    ```bash
    cd ~/robot_ws/src
    git pull origin main
    ```

3.  **Install System Dependencies:**
    ```bash
    cd ~/robot_ws
    rosdep install --from-paths src --ignore-src -r -y
    ```

4.  **Build the Workspace:**
    ```bash
    cd ~/robot_ws
    colcon build --symlink-install
    ```

5.  **Source the Environment:**
    ```bash
    source ~/.bashrc
    ```

## Usage Procedures

### Launching the Simulation Environment
To initialize the virtual robot within the Gazebo physics engine:
```bash
ros2 launch esibot_gazebo sim.launch.py
```

### Visualizing the Kinematic Model
To inspect the robot's geometry, verify TF tree transformations, or test hardware representations without the computational overhead of Gazebo physics. This configuration loads the URDF, publishes transformations, and supports Foxglove bridge integration for browser-based visualization:
```bash
ros2 launch esibot_description display.launch.py
```
